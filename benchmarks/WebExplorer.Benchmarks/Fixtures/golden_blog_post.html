Building a REST API with ASP.NET Core

By Alex Tech September 20, 2025 10 min read

Building a REST API is one of the most common tasks for backend developers. In this guide, we will walk through creating a production-ready API using ASP.NET Core 10.

Prerequisites

Before we begin, make sure you have the following installed:

  • .NET 10 SDK
  • Visual Studio 2025 or VS Code
  • SQL Server or PostgreSQL

Project Setup

Start by creating a new web API project:

dotnet new webapi -n MyApi
cd MyApi

This creates a minimal API template with a weather forecast endpoint. We will replace this with our own domain model.

Defining the Model

For this tutorial, we will build a simple task management API. Let's define our model:

public class TodoItem
{
    public int Id { get; set; }
    public string Title { get; set; } = "";
    public bool IsComplete { get; set; }
    public DateTime CreatedAt { get; set; }
}

Setting Up Entity Framework Core

Add the Entity Framework Core packages:

dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Design

Create the database context and configure the connection in Program.cs.

Creating the API Endpoints

With minimal APIs, defining endpoints is straightforward:

app.MapGet("/api/todos", async (TodoDb db) =>
    await db.Todos.ToListAsync());

app.MapPost("/api/todos", async (TodoItem todo, TodoDb db) =>
{
    db.Todos.Add(todo);
    await db.SaveChangesAsync();
    return Results.Created($"/api/todos/{todo.Id}", todo);
});

Adding Validation

Input validation is critical for security. Use FluentValidation or data annotations to ensure all input meets your requirements.

Never trust user input. Always validate on the server side, even if you have client-side validation.

Error Handling

Implement global error handling with middleware to return consistent error responses:

app.UseExceptionHandler("/error");
app.Map("/error", (HttpContext context) =>
{
    return Results.Problem();
});

Conclusion

ASP.NET Core makes it easy to build performant, production-ready APIs. The minimal API approach reduces boilerplate while maintaining full access to the framework's capabilities.
